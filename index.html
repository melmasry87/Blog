Tips & Tricks

Quick Array Deduplication

Remove duplicates from arrays in JavaScript with this one-liner:

const unique = [...new Set(array)];

This uses the Set object which only stores unique values, then spreads it back into an array.

Git Stash with Message

Save your work in progress with a descriptive message:

git stash push -m "work in progress on feature X"

Makes it easier to identify what you stashed when you have multiple stashes.

Python List Comprehension Power

Filter and transform data in one line:

squares = [x**2 for x in range(10) if x % 2 == 0]

Gets squares of even numbers from 0-9. More readable and faster than loops.

VS Code Multi-Cursor Magic

Edit multiple lines simultaneously:

Ctrl+Alt+Arrow Keys (Windows/Linux) Cmd+Option+Arrow Keys (Mac)

Perfect for refactoring variable names or making similar edits across multiple lines.

Optional Chaining Operator

Safely access nested object properties:

const city = user?.address?.city ?? 'Unknown';

No more "Cannot read property of undefined" errors when accessing deep object properties.

Interactive Rebase

Clean up your commit history before pushing:

git rebase -i HEAD~3

Lets you squash, reorder, or edit your last 3 commits for a cleaner project history.

PowerShell Object Pipeline

Filter and sort processes by memory usage:

Get-Process | Where-Object {$_.WorkingSet -gt 100MB} | Sort-Object WorkingSet -Descending

PowerShell's object-based pipeline makes data manipulation incredibly powerful and readable.

SQL Window Functions

Rank records within groups without subqueries:

SELECT name, salary, RANK() OVER (PARTITION BY department ORDER BY salary DESC) as rank FROM employees;

Window functions provide powerful analytical capabilities for complex queries.

PowerShell Splatting

Pass multiple parameters cleanly using hashtables:

$params = @{ Path = "C:\temp" Recurse = $true Filter = "*.log" } Get-ChildItem @params

Makes commands with many parameters more readable and maintainable.

SQL Common Table Expressions

Create readable recursive or complex queries:

WITH RECURSIVE factorial(n, result) AS ( SELECT 1, 1 UNION ALL SELECT n+1, result*(n+1) FROM factorial WHERE n < 5 ) SELECT * FROM factorial;

CTEs make complex queries more readable and enable recursive operations.
